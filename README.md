<h2>۱. معرفی پروژه</h2>

این پروژه با هدف طراحی و پیاده‌سازی یک پنل مدیریت تحت وب برای سامانه هوشمند کشاورزی «هاویرکشت» انجام شده است. پنل مدیریت به‌صورت یک برنامه تک‌صفحه‌ای پیاده‌سازی شده و وظیفه آن فراهم‌کردن امکان مدیریت داده‌های پایه سامانه از طریق ارتباط مستقیم با سرویس‌های سمت سرور است.

در این پروژه تلاش شده است مفاهیم اصلی درس برنامه‌نویسی وب به‌صورت عملی و مطابق با ساختارهای واقعی پروژه‌های نرم‌افزاری پیاده‌سازی شود و ارتباط بین رابط کاربری و API به‌طور کامل برقرار گردد.


<h2>۲. اهداف پروژه</h2>

اهداف این پروژه مطابق با فایل ارائه‌شده توسط استاد درس تعریف شده و شامل موارد زیر است:

آشنایی مقدماتی با سرویس‌های سمت سرور و مستندات Swagger  
آشنایی با نحوه طراحی و پیاده‌سازی صفحات سمت کاربر  
برقراری ارتباط بین Frontend و Backend از طریق API  
آشنایی با مفاهیم احراز هویت و مدیریت دسترسی کاربران  
آشنایی با مدیریت Routeها در برنامه‌های تک‌صفحه‌ای  
آشنایی با ساختار پروژه‌های مبتنی بر React  
استفاده از مدل‌های پاسخ (Response Model) در ارتباط با API  
آشنایی با مفاهیم Query Parameter، Path Parameter و Request Body  
آشنایی با استفاده از Git و ساختار پروژه‌های نرم‌افزاری  
پیاده‌سازی یک نمونه واقعی از پنل مدیریت تحت وب


<h2>۳. تکنولوژی‌ها و ابزارهای استفاده‌شده</h2>

در پیاده‌سازی این پروژه از ابزارها و فناوری‌های زیر استفاده شده است:

در سمت کاربر:
React نسخه ۱۹  
React Router DOM برای مدیریت مسیرها  
TailwindCSS برای طراحی رابط کاربری  
Axios برای ارتباط با API  
Formik برای مدیریت فرم‌ها  
Yup برای اعتبارسنجی داده‌های ورودی  

در سمت سرور (در قالب API آماده):
FastAPI  
JWT برای احراز هویت  
Swagger / OpenAPI برای مستندسازی API  


<h2>۴. ساختار کلی پروژه</h2>

پروژه به‌صورت یک برنامه تک‌صفحه‌ای طراحی شده و از معماری ماژولار استفاده می‌کند. ساختار کلی پروژه شامل بخش‌های زیر است:

صفحات احراز هویت  
صفحات مدیریتی  
Layout اصلی پنل مدیریت  
Route Guard برای محافظت از مسیرها  
لایه ارتباط با API  

تمام صفحات مدیریتی درون یک Layout مشترک نمایش داده می‌شوند و تنها کاربران احراز هویت‌شده امکان دسترسی به آن‌ها را دارند.

ساختار کلی

<h2>۵. پیاده‌سازی احراز هویت کاربران</h2>

برای ورود کاربران به سامانه، یک صفحه ورود طراحی شده است که شامل فرم نام کاربری و رمز عبور می‌باشد. مدیریت فرم ورود با استفاده از Formik انجام شده و اعتبارسنجی فیلدها توسط Yup صورت گرفته است.

پس از ارسال اطلاعات ورود، درخواست به سرویس احراز هویت ارسال شده و در صورت موفقیت، توکن‌های دسترسی و نوسازی دریافت می‌شوند. این توکن‌ها در حافظه مرورگر ذخیره شده و برای دسترسی به مسیرهای محافظت‌شده مورد استفاده قرار می‌گیرند.

این بخش امکان مدیریت وضعیت ورود کاربر و کنترل دسترسی به بخش‌های مختلف پنل را فراهم می‌کند.


<h2>۶. محافظت از مسیرها (Route Guard)</h2>

برای جلوگیری از دسترسی کاربران غیرمجاز، از یک Route Guard استفاده شده است. این مکانیزم بررسی می‌کند که آیا کاربر دارای توکن معتبر است یا خیر.

در صورت وجود توکن، دسترسی به صفحات مدیریتی مجاز خواهد بود و در غیر این صورت، کاربر به صفحه ورود هدایت می‌شود. این ساختار با استفاده از قابلیت مسیرهای تو در تو در React Router پیاده‌سازی شده است.


<h2>۷. طراحی Layout پنل مدیریت</h2>

پنل مدیریت دارای یک Layout ثابت است که شامل بخش‌های زیر می‌باشد:

هدر  
سایدبار  
محتوای اصلی صفحه  

تمام صفحات مدیریتی در داخل این Layout نمایش داده می‌شوند. طراحی رابط کاربری به‌صورت راست‌به‌چپ انجام شده و از TailwindCSS برای پیاده‌سازی ظاهر پنل استفاده شده است.

این ساختار باعث یکپارچگی ظاهر صفحات و سهولت در توسعه بخش‌های جدید می‌شود.


<h2>۸. مدیریت استان‌ها</h2>

یکی از بخش‌های اصلی پیاده‌سازی‌شده در این پروژه، بخش مدیریت استان‌ها است. این بخش امکان مشاهده لیست استان‌ها را فراهم می‌کند و شامل قابلیت‌های زیر است:

دریافت لیست استان‌ها از سرور  
نمایش اطلاعات در قالب جدول  
جستجوی استان‌ها  
صفحه‌بندی داده‌ها  
امکان حذف استان  

اطلاعات استان‌ها از طریق API و مطابق با مستندات OpenAPI دریافت می‌شود و ارتباط بین رابط کاربری و سرویس سمت سرور به‌صورت کامل برقرار است.


<h2>۹. ارتباط با API و استفاده از Swagger</h2>

تمام ارتباطات با Backend بر اساس مستندات Swagger انجام شده است. برای هر سرویس، نوع درخواست، پارامترها و ساختار پاسخ‌ها مطابق با فایل OpenAPI در نظر گرفته شده است.

در این پروژه از انواع مختلف پارامترها شامل Query Parameter، Path Parameter و Request Body استفاده شده و نحوه ارسال و دریافت داده‌ها به‌صورت عملی پیاده‌سازی شده است.


<h2>۱۰. سخن پایانی</h2>

این پروژه نمونه‌ای کامل از پیاده‌سازی عملی مفاهیم مطرح‌شده در درس برنامه‌نویسی وب است. در این پروژه، ارتباط بین Frontend و Backend به‌صورت واقعی برقرار شده و ساختار یک پنل مدیریت استاندارد پیاده‌سازی شده است.

متاسفانه پروژه به دلیل اتفاقات اخیر در کشور، قطعی اینترنت و سامانه در وقت باقی مانده تکمیل نشده اما پروژه قابلیت توسعه برای افزودن بخش‌های جدید مانند مدیریت شهرها، روستاها و سایر داده‌های سامانه را دارد و چرا که به شکل استاندارد توسعه داده شده و تلاش شده مفاهیمی که در این درس یاد گرفته شده را تا آنجایی که راه دارد پیاده سازی کرده و نشان داده شود که مفاهیم و آموزش ها مطالعه و پیاده سازی شده است.
